Паттерн проектирования(design pattern -шаблон не путать с шаблоном страницы) - архитектурная, представляющая собой решение
проблемы проектирования.
Виды паттернов:
FrontController если  разделов мало и сайт небольшой и нельзя использовать ЧПУ т.к обращаемся напрямую
MVC
	|-frontController index.php  настроив наш сервер определённым образом ,запросы будут передаваться на frontController
		здесь будут:
		*соединение с базой данных
		*Создан компонент router которы перенаправит на нужную страницу
		и основываясь на определённых правилах
		он будет решать как отображать страницу.
		|-file1 /news/
		|-file2 /about/
		|-file3 /register/

M -model  манипулирует данными,реализует бизнес-логику(работа с данными), часто реализуется в виде классов
V -view (Представление,визуализация),отображение товаров плиткой списком и прочее, хотя данные одинаковы.
C -controller  на основе данных из запроса, получает данные из модели, и использует данные из представления ,после отображает необходимым образом
Контроллеров может быть много
